btk_link_button_set_uri_hook ()

BtkLinkButtonUriFunc
btk_link_button_set_uri_hook (BtkLinkButtonUriFunc func,
                              gpointer data,
                              GDestroyNotify destroy);

btk_link_button_set_uri_hook has been deprecated since version 2.24 and should not be used in newly-written code.

Use the “clicked” signal instead

Sets func as the function that should be invoked every time a user clicks a BtkLinkButton. This function is called before every callback registered for the "clicked" signal.

If no uri hook has been set, BTK+ defaults to calling btk_show_uri().

Parameters

func

a function called each time a BtkLinkButton is clicked, or NULL.

[allow-none]

data

user data to be passed to func , or NULL.

[allow-none]

destroy

a GDestroyNotify that gets called when data is no longer needed, or NULL.

[allow-none]

Returns

the previously set hook function.

Since: 2.10

btk_link_button_get_visited ()

gboolean
btk_link_button_get_visited (BtkLinkButton *link_button);

Retrieves the 'visited' state of the URI where the BtkLinkButton points. The button becomes visited when it is clicked. If the URI is changed on the button, the 'visited' state is unset again.

The state may also be changed using btk_link_button_set_visited().

Parameters

link_button

a BtkLinkButton

  

Returns

TRUE if the link has been visited, FALSE otherwise

Since: 2.14
