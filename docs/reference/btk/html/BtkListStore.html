btk_list_store_new ()

BtkListStore *
btk_list_store_new (bint n_columns,
                    ...);

Creates a new list store as with n_columns columns each of the types passed in. Note that only types derived from standard BObject fundamental types are supported.

As an example, btk_tree_store_new (3, B_TYPE_INT, B_TYPE_STRING, BDK_TYPE_PIXBUF); will create a new BtkListStore with three columns, of type int, string and BdkPixbuf respectively.

btk_list_store_set ()

void
btk_list_store_set (BtkListStore *list_store,
                    BtkTreeIter *iter,
                    ...);

Sets the value of one or more cells in the row referenced by iter . The variable argument list should contain integer column numbers, each column number followed by the value to be set. The list is terminated by a -1. For example, to set column 0 with type B_TYPE_STRING to "Foo", you would write btk_list_store_set (store, iter, 0, "Foo", -1).

The value will be referenced by the store if it is a B_TYPE_OBJECT, and it will be copied if it is a B_TYPE_STRING or B_TYPE_BOXED.

btk_list_store_insert_with_values ()

void
btk_list_store_insert_with_values (BtkListStore *list_store,
                                   BtkTreeIter *iter,
                                   bint position,
                                   ...);

Creates a new row at position . iter will be changed to point to this new row. If position is larger than the number of rows on the list, then the new row will be appended to the list. The row will be filled with the values given to this function.

Calling btk_list_store_insert_with_values(list_store, iter, position...) has the same effect as calling 

1
2
 
btk_list_store_insert (list_store, iter, position);
btk_list_store_set (list_store, iter, ...);

with the difference that the former will only emit a row_inserted signal, while the latter will emit row_inserted, row_changed and, if the list store is sorted, rows_reordered. Since emitting the rows_reordered signal repeatedly can affect the performance of the program, btk_list_store_insert_with_values() should generally be preferred when inserting rows in a sorted list store.
