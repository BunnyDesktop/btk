btk_recent_manager_new ()

BtkRecentManager *
btk_recent_manager_new (void);

Creates a new recent manager object. Recent manager objects are used to handle the list of recently used resources. A BtkRecentManager object monitors the recently used resources list, and emits the "changed" signal each time something inside the list changes.

BtkRecentManager objects are expensive: be sure to create them only when needed. You should use btk_recent_manager_get_default() instead.

Returns

A newly created BtkRecentManager object.

Since: 2.10

Gets the recent manager object associated with screen ; if this function has not previously been called for the given screen, a new recent manager object will be created and associated with the screen. Recent manager objects are fairly expensive to create, so using this function is usually a better choice than calling btk_recent_manager_new() and setting the screen yourself; by using this function a single recent manager object will be shared between users.

btk_recent_manager_add_full ()

bboolean
btk_recent_manager_add_full (BtkRecentManager *manager,
                             const bchar *uri,
                             const BtkRecentData *recent_data);

Adds a new resource, pointed by uri , into the recently used resources list, using the metadata specified inside the BtkRecentData structure passed in recent_data .

The passed URI will be used to identify this resource inside the list.

In order to register the new recently used resource, metadata about the resource must be passed as well as the URI; the metadata is stored in a BtkRecentData structure, which must contain the MIME type of the resource pointed by the URI; the name of the application that is registering the item, and a command line to be used when launching the item.

Optionally, a BtkRecentData structure might contain a UTF-8 string to be used when viewing the item instead of the last component of the URI; a short description of the item; whether the item should be considered private - that is, should be displayed only by the applications that have registered it.

Parameters
