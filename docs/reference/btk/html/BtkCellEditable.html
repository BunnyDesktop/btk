Signal Details

The “editing-done” signal

void
user_function (BtkCellEditable *cell_editable,
               bpointer         user_data)

This signal is a sign for the cell renderer to update its value from the cell_editable .

Implementations of BtkCellEditable are responsible for emitting this signal when they are done editing, e.g. BtkEntry is emitting it when the user presses Enter.

btk_cell_editable_editing_done() is a convenience method for emitting BtkCellEditable::editing-done.

The “remove-widget” signal

void
user_function (BtkCellEditable *cell_editable,
               bpointer         user_data)

This signal is meant to indicate that the cell is finished editing, and the widget may now be destroyed.

Implementations of BtkCellEditable are responsible for emitting this signal when they are done editing. It must be emitted after the “editing-done” signal, to give the cell renderer a chance to update the cell's value before the widget is removed.

btk_cell_editable_remove_widget() is a convenience method for emitting BtkCellEditable::remove-widget.

Parameters

cell_editable

the object on which the signal was emitted
