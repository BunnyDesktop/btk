New features

BtkFileChooser, starting with BTK+ 2.4, is the new set of APIs for file selection widgets and dialogs. Previous versions of BTK+ used BtkFileSelection, which has numerous problems.

BtkFileChooser is an abstract interface that can be implemented by widgets that perform file selection tasks. Two widgets in BTK+ implement this interface: BtkFileChooserDialog and BtkFileChooserWidget. Most applications simply need to use BtkFileChooserDialog, which is a dialog box that allows the user to select existing files for opening them, or to pick new filenames for saving documents. BtkFileChooserWidget is for special applications that need to embed a file selection widget inside a larger window. In the context of BTK+, BtkFileChooserDialog is simply a BtkDialog box with a BtkFileChooserWidget. inside.

Creating a BtkFileChooserDialog

To create a BtkFileChooserDialog, you simply call btk_file_chooser_dialog_new(). This function is similar to btk_dialog_new() in that it takes parameters for the title of the dialog box and its transient parent, as well as its buttons. In addition, it takes in an argument that determines whether the file chooser dialog will be used for opening existing files or for saving to a possibly new file.
