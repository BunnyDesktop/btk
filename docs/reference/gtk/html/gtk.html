Part I. BTK+ Overview

BTK+ is a library for creating graphical user interfaces. It works on many UNIX-like platforms, Windows, and on framebuffer devices. BTK+ is released under the GNU Library General Public License (GNU LGPL), which allows for flexible licensing of client applications. BTK+ has a C-based object-oriented architecture that allows for maximum flexibility. Bindings for other languages have been written, including C++, Objective-C, Guile/Scheme, Perl, Python, TOM, Ada95, Free Pascal, and Eiffel.

BTK+ depends on the following libraries:

GLib

A general-purpose utility library, not specific to graphical user interfaces. GLib provides many useful data types, macros, type conversions, string utilities, file utilities, a main loop abstraction, and so on.

Bango

Bango is a library for internationalized text handling. It centers around the #BangoLayout object, representing a paragraph of text. Bango provides the engine for #BtkTextView, #BtkLabel, #BtkEntry, and other widgets that display text.

BATK

BATK is the Accessibility Toolkit. It provides a set of generic interfaces allowing accessibility technologies to interact with a graphical user interface. For example, a screen reader uses BATK to discover the text in an interface and read it to blind users. BTK+ widgets have built-in support for accessibility using the BATK framework.

BdkPixbuf

This is a small library which allows you to create #BdkPixbuf ("pixel buffer") objects from image data or image files. Use a #BdkPixbuf in combination with #BtkImage to display images.

BDK

BDK is the abstraction layer that allows BTK+ to support multiple windowing systems. BDK provides drawing and window system facilities on X11, Windows, and the Linux framebuffer device.

BTK+

The BTK+ library itself contains widgets, that is, GUI components such as #BtkButton or #BtkTextView.
