bdk_rgb_find_color ()

void
bdk_rgb_find_color (BdkColormap *colormap,
                    BdkColor *color);

bdk_rgb_find_color has been deprecated since version 2.22 and should not be used in newly-written code.

Bairo handles colors automatically.

colormap should be the colormap for the graphics context and drawable you're using to draw. If you're drawing to a BtkWidget, call btk_widget_get_colormap().

color should have its red, green, and blue fields initialized; bdk_rgb_find_color() will fill in the pixel field with the best matching pixel from a color cube. The color is then ready to be used for drawing, e.g. you can call bdk_gc_set_foreground() which expects pixel to be initialized.

In many cases, you can avoid this whole issue by calling bdk_gc_set_rgb_fg_color() or bdk_gc_set_rgb_bg_color(), which do not expect pixel to be initialized in advance. If you use those functions, there's no need for bdk_rgb_find_color().

bdk_rgb_get_visual ()

BdkVisual *
bdk_rgb_get_visual (void);

bdk_rgb_get_visual has been deprecated since version 2.22 and should not be used in newly-written code.

Use bdk_visual_get_system_visual (bdk_screen_get_default()) instead.

Gets a "preferred visual" chosen by BdkRGB for rendering image data on the default screen. In previous versions of BDK, this was the only visual BdkRGB could use for rendering. In current versions, it's simply the visual BdkRGB would have chosen as the optimal one in those previous versions. BdkRGB can now render to drawables with any visual.

Returns

The BdkVisual chosen by BdkRGB.

[transfer none]

bdk_rgb_get_colormap ()

BdkColormap *
bdk_rgb_get_colormap (void);

bdk_rgb_get_colormap has been deprecated since version 2.22 and should not be used in newly-written code.

Use bdk_screen_get_system_colormap (bdk_screen_get_default()) instead.

Get the preferred colormap for rendering image data. Not a very useful function; historically, BDK could only render RGB image data to one colormap and visual, but in the current version it can render to any colormap and visual. So there's no need to call this function.

Returns

the preferred colormap.
